C · Architecture

How Astro, Firebase & Razorpay fit together

Astro renders the pages and exposes a few server endpoints. The secret-key work (verifying payment, writing bookings) never happens in the browser.

bookings { client, session, datetime, status }
Firebase Auth

Razorpay
Order + checkout + signed callback
Safety: the browser only ever holds the public Razorpay key. The secret key, signature verification, and the "mark booking paid" write all live on the Astro server. A faked "payment success" from the client is rejected because the signature won't match.
